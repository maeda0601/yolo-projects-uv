# YOLO 画像収集・学習 GUI アプリケーション

YOLOv8を使用した物体検出モデルの学習をGUIで行えるアプリケーションです。
画像収集、アノテーション、学習までの一連の作業をこのアプリ1つで完結できます。

## 機能

- **画像収集**: カメラ撮影またはファイルインポートによる画像収集
- **アノテーション**: マウス操作によるバウンディングボックスの描画・編集
- **学習**: YOLOv8モデルの学習実行と進捗確認
- **クラス管理**: 検出クラスの追加・編集・削除

## 動作環境

- Python 3.8以上
- Windows / macOS / Linux

## インストール手順

### 1. リポジトリのクローン

```bash
git clone https://github.com/your-username/yolo-projects.git
cd yolo-projects
```

### 2. 依存パッケージのインストール（uv使用）

[uv](https://docs.astral.sh/uv/) を使用した環境構築を推奨します。

```bash
uv sync
```

仮想環境（`.venv/`）の作成とパッケージのインストールが自動で行われます。

> **uvがインストールされていない場合:**
> ```bash
> # Windows (PowerShell)
> powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
> ```

### 3. アプリケーションの起動

```bash
uv run python gui_app.py
```

## 使い方

### 画像収集タブ

学習に使用する画像を収集します。

#### カメラ撮影

1. 「カメラID」に使用するカメラの番号を入力（通常は0）
2. 「カメラ開始」ボタンをクリック
3. プレビュー画面でカメラ映像を確認
4. 「保存先」でTrain/Valを選択
5. 「ファイル名プレフィックス」を設定
6. 「撮影」ボタンで画像を保存

#### ファイルインポート

1. 「画像ファイルを選択」ボタンをクリック
2. インポートする画像を選択（複数選択可）
3. 選択した画像がdatasetフォルダにコピーされます

### アノテーションタブ

収集した画像にバウンディングボックス（BBox）を描画します。

#### 基本操作

1. 左側のリストから画像を選択
2. 右側の「クラス選択」で描画するクラスを選択
3. 画像上で左ドラッグしてBBoxを描画
4. 「アノテーション保存」ボタンで保存

#### 操作方法

| 操作 | 説明 |
|------|------|
| 左ドラッグ | BBoxを描画 |
| 右クリック | クリック位置のBBoxを削除 |
| 次の画像へ | 保存して次の画像に移動 |

#### フィルター機能

- **全て**: すべての画像を表示
- **未ラベル**: アノテーションがない画像のみ表示
- **ラベル済**: アノテーション済みの画像のみ表示

### 学習タブ

YOLOv8モデルの学習を実行します。

#### 学習設定

| 項目 | 説明 | デフォルト値 |
|------|------|--------------|
| モデルサイズ | YOLOv8のモデルサイズ（n/s/m/l/x） | yolov8n |
| エポック数 | 学習の繰り返し回数 | 100 |
| バッチサイズ | 1回の学習で使用する画像数 | 16 |
| 画像サイズ | 入力画像のサイズ | 640 |
| 事前学習モデル | 転移学習に使用するモデル | yolov8n.pt |
| プロジェクト名 | 結果の保存先フォルダ名 | model |
| 実験名 | 実験のサブフォルダ名 | exp |
| デバイス | 使用するデバイス（GPU番号 or cpu） | 0 |

#### 学習の実行

1. 各パラメータを設定
2. 「学習開始」ボタンをクリック
3. 学習ログで進捗を確認
4. 学習完了後、`model/exp/weights/`に学習済みモデルが保存されます

### クラス管理タブ

検出するクラス（カテゴリ）を管理します。

#### クラスの追加

1. 「クラス名」にクラス名を入力
2. 「クラス追加」ボタンをクリック

#### クラスの編集・削除

1. リストからクラスを選択
2. 「クラス名を編集」または「選択したクラスを削除」をクリック

#### 設定の保存

「data.yaml保存」ボタンでクラス情報を保存します。

## ディレクトリ構造

```
yolo-projects-uv/
├── gui_app.py          # 画像収集・アノテーション・学習GUIアプリ
├── camera_app.py       # リアルタイム物体検出・異常検知アプリ
├── pyproject.toml      # プロジェクト設定・依存パッケージ定義（uv）
├── uv.lock             # 依存パッケージのバージョン固定ファイル
├── requirements.txt    # 旧依存ファイル（互換参照用）
├── data.yaml           # データセット設定ファイル（自動生成）
├── demo/               # サンプルスクリプト
│   ├── demo.py         # 最小構成の検出デモ
│   └── camera_app.py   # camera_app のデモ用コピー
├── dataset/            # データセットフォルダ（自動生成）
│   ├── images/
│   │   ├── train/      # 学習用画像
│   │   └── val/        # 検証用画像
│   └── labels/
│       ├── train/      # 学習用ラベル（YOLO形式）
│       └── val/        # 検証用ラベル（YOLO形式）
└── model/              # 学習結果の保存先（自動生成）
    └── exp/
        └── weights/
            ├── best.pt     # 最良モデル
            └── last.pt     # 最終モデル
```

## リアルタイム検出アプリ（camera_app.py）

学習済みモデルを使用してカメラ映像をリアルタイムに物体検出します。

```bash
uv run python camera_app.py
```

- 学習済みモデル（`.pt` ファイル）をアプリ内で選択
- 信頼度しきい値をスライダーで調整
- 検出結果が正常状態から外れると異常アラートを表示

## data.yaml の形式

```yaml
path: ./dataset
train: images/train
val: images/val
names:
  0: class_name_1
  1: class_name_2
```

## YOLO ラベル形式

各画像に対応するラベルファイル（.txt）は以下の形式です：

```
<class_id> <x_center> <y_center> <width> <height>
```

- すべての値は画像サイズで正規化（0.0〜1.0）
- 1行に1つのオブジェクト

例：
```
0 0.500000 0.500000 0.200000 0.300000
1 0.750000 0.250000 0.150000 0.200000
```

## トラブルシューティング

### カメラが認識されない

- カメラIDを変更してみてください（0, 1, 2...）
- 他のアプリがカメラを使用していないか確認してください

### 学習が開始されない

- `ultralytics`が正しくインストールされているか確認してください
- GPUを使用する場合、CUDAが正しくインストールされているか確認してください
- CPUで学習する場合は、デバイスを「cpu」に設定してください

### メモリエラーが発生する

- バッチサイズを小さくしてください（16 → 8 → 4）
- 画像サイズを小さくしてください（640 → 416 → 320）
- モデルサイズを小さくしてください（yolov8n）
